### 冲突详情

#### 1. R1 与 C1：实现顺序冲突
- **冲突**: 先实现 R1 还是先提取公共逻辑？
- **决策**: ✅ 已确定先实现 R1，再抽象

#### 2. R1 与 C2：不足单元模式应用层级 ✅ 已确认
- **问题**: CompositeTimeRule 有两层（相对时段、自然时段），不足单元模式应用在哪一层？
- **确认结果**:
  - 内层（自然时段边界）：应用不足单元判断（FULL/PROPORTIONAL）
  - 外层（相对时间段边界）：直接截断，截断部分按不足单元判断（FULL/PROPORTIONAL）
- **配置位置**: 规则级别配置，应用于两种边界

#### 3. R1 与 P3：双层封顶 vs 周期延长 ✅ 已确认
- **问题**: CompositeTimeRule 有时间段封顶和周期封顶，周期延长型免费时段如何影响？
- **场景**:
  ```
  时间段1 (0-120分钟): 独立封顶 5元
  时间段2 (120-1440分钟): 无独立封顶
  周期封顶: 50元
  免费时段 9:00-10:00（周期延长型）
  ```
- **确认结果**: 采用"气泡弹开"模型
  - 免费时段像水管中的气泡，将计费时间"弹开"
  - 气泡前的段：边界不变
  - 气泡后的段：边界整体后移 = 气泡长度
  - 详细设计见 `2026-03-18-composite-time-rule-design.md` 第四节

#### 4. R1 与 O1：长期简化与双层结构 ✅ 已确认
- **问题**: CompositeTimeRule 的双层结构如何简化？
- **确认结果**:
  - 简化金额 = 周期封顶金额
  - 前提条件：该周期内无任何优惠
  - 实现逻辑：逐周期检查是否有优惠，无优惠的连续周期可合并简化
- **待办特性**: 规律性优惠的简化计算（如每天固定免费2小时）

#### 5. P3 与 O1：周期延长与长期简化 ✅ 已确认
- **问题**: 周期延长后，长期简化的周期数如何计算？
- **确认结果**:
  - 周期延长型免费时段本身就是优惠
  - 第一版实现：有优惠就不能简化，必须逐周期计算
  - 所以第一版实现中此问题不存在冲突

#### 6. P4 与 R1：时长扣除等效金额计算
- **问题**: CompositeTimeRule 有多个单价，DEDUCT_DURATION 如何计算等效金额？
- **选项**:
  - A: 取平均单价
  - B: 取最低单价
  - C: 取规则配置的基准单价
  - D: 按当前周期实际使用的单价加权平均
