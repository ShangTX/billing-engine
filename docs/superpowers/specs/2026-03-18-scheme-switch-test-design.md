# 方案切换场景测试设计

**Status:** 🚧 设计中

**Date:** 2026-03-18

---

## 业务场景

景区停车场收费：
- **旺季**：4月20日 - 10月10日，单价高、单元短
- **淡季**：10月11日 - 次年4月19日，单价低、单元长

计费跨越淡旺季边界时，方案切换。

---

## 测试维度

### 维度1：切换次数

| 类型 | 描述 | 场景 |
|------|------|------|
| 无切换 | 单方案整个周期 | 旺季期间或淡季期间 |
| 单次切换 | 旺季→淡季 或 淡季→旺季 | 跨越一个边界 |
| 多次切换 | 跨越多个边界 | 长期停车（数月） |

### 维度2：分段计算模式

| 模式 | 说明 | 预期行为 |
|------|------|---------|
| SEGMENT_LOCAL | 分段独立起算 | 每个分段从自己的起点计算，周期重置 |
| GLOBAL_ORIGIN | 全局起算截取 | 从全局起点计算，分段只截取结果 |

### 维度3：计费模式

| 模式 | 说明 |
|------|------|
| UNIT_BASED | 固定单元长度，不因优惠切分 |
| CONTINUOUS | 按免费时段边界切分 |

### 维度4：CONTINUE模式

| 场景 | 说明 |
|------|------|
| 切换前起点 | 从上次计算结果继续，跨方案边界 |
| 切换后起点 | 在新方案内继续计算 |

## 测试场景设计

### 场景1：单次切换 - 旺季→淡季（SEGMENT_LOCAL）

```
计费时间：10月01日 08:00 - 10月15日 08:00
切换点：10月11日 00:00

分段：
- 分段1：10月01日 08:00 - 10月11日 00:00（旺季方案）
- 分段2：10月11日 00:00 - 10月15日 08:00（淡季方案）

SEGMENT_LOCAL：
- 分段1：从10月01日 08:00 起算，周期边界 10月02日 08:00
- 分段2：从10月11日 00:00 起算，周期边界 10月12日 00:00
```

### 场景2：单次切换 - 淡季→旺季（GLOBAL_ORIGIN）

```
计费时间：04月15日 08:00 - 04月25日 08:00
切换点：04月20日 00:00

分段：
- 分段1：04月15日 08:00 - 04月20日 00:00（淡季方案）
- 分段2：04月20日 00:00 - 04月25日 08:00（旺季方案）

GLOBAL_ORIGIN：
- 全局起点：04月15日 08:00
- 分段1：从全局起点计算，截取 04月15日-04月20日
- 分段2：从全局起点计算，截取 04月20日-04月25日
```

### 场景3：多次切换 - 跨三个季节

```
计费时间：09月01日 08:00 - 次年05月01日 08:00
切换点：10月11日 00:00（旺季→淡季）、04月20日 00:00（淡季→旺季）

分段：
- 分段1：09月01日 08:00 - 10月11日 00:00（旺季）
- 分段2：10月11日 00:00 - 04月20日 00:00（淡季）
- 分段3：04月20日 00:00 - 05月01日 08:00（旺季）
```

### 场景4：CONTINUE模式跨方案切换

```
第一次计算：10月01日 08:00 - 10月05日 08:00（旺季）
第二次计算：CONTINUE 到 10月15日 08:00（跨入淡季）

预期：
- 第二次计算自动识别方案切换
- 分段1继续：10月05日 - 10月11日（旺季）
- 分段2新起：10月11日 - 10月15日（淡季）
```

### 场景5：切换点在计费单元中间

```
计费时间：10月10日 22:00 - 10月11日 02:00
切换点：10月11日 00:00

单元长度：60分钟
分段1最后单元：10月10日 22:00 - 10月10日 23:00（旺季）
分段1最后单元：10月10日 23:00 - 10月11日 00:00（旺季）
分段2第一单元：10月11日 00:00 - 10月11日 01:00（淡季）
分段2第二单元：10月11日 01:00 - 10月11日 02:00（淡季）

验证：切换点恰好是单元边界，不产生截断单元
```

## 规则配置

### 旺季方案 (peak-season)

```java
RelativeTimeConfig.builder()
    .id("peak-rule")
    .periods(List.of(
        RelativeTimePeriod.builder()
            .beginMinute(0)
            .endMinute(1440)
            .unitMinutes(30)      // 30分钟单元
            .unitPrice(new BigDecimal("2"))  // 2元/单元
            .build()
    ))
    .maxChargeOneCycle(new BigDecimal("50"))  // 日封顶50元
    .build()
```

### 淡季方案 (off-season)

```java
RelativeTimeConfig.builder()
    .id("off-peak-rule")
    .periods(List.of(
        RelativeTimePeriod.builder()
            .beginMinute(0)
            .endMinute(1440)
            .unitMinutes(60)      // 60分钟单元
            .unitPrice(new BigDecimal("1"))  // 1元/单元
            .build()
    ))
    .maxChargeOneCycle(new BigDecimal("20"))  // 日封顶20元
    .build()
```

## 验证要点

### 1. 分段正确性

- [ ] 分段时间边界正确
- [ ] 每个分段使用正确的方案ID
- [ ] 分段ID唯一且有序

### 2. SEGMENT_LOCAL 模式

- [ ] 每个分段独立计算周期
- [ ] 每个分段封顶独立
- [ ] 分段金额正确

### 3. GLOBAL_ORIGIN 模式

- [ ] 全局起点正确传递
- [ ] 周期边界从全局起点计算
- [ ] 封顶跨分段累计（待确认）

### 4. CONTINUE 模式

- [ ] 方案切换点自动识别
- [ ] carryOver 正确传递到对应分段
- [ ] 跨方案状态正确恢复

### 5. 边界情况

- [ ] 切换点恰好是单元边界
- [ ] 切换点在单元中间（截断单元处理）
- [ ] 零长度分段（切换点等于计费边界）
