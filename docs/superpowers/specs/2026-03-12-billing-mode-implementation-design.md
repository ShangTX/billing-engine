# 计费模式差异化实现设计

## 概述

在 DayNightRule 和 RelativeTimeRule 中根据 `billingMode` 实现不同的计算方式，支持 UNIT_BASED 和 CONTINUOUS 两种模式。

## 模式行为对比

### UNIT_BASED（计费单位模式）

```
场景：停车 05:00 - 08:15，单元60分钟，免费时段 06:30-07:30

05:00-06:00  2元  (完整单元)
06:00-07:00  2元  (免费时段未完全覆盖单元 → 收费)
07:00-08:00  2元  (免费时段未完全覆盖单元 → 收费)
08:00-08:15  2元  (不足单元，收全额)
─────────────────
总计：8元
```

**特点**：
- 时间按固定单元对齐（从计费起点开始）
- 免费时段必须完全覆盖整个单元才免费
- 不足一个单元也收全额
- 封顶时按比例削减每个收费单元

### CONTINUOUS（连续时间模式）

```
场景：停车 05:00 - 08:15，单元60分钟，免费时段 06:30-07:30

05:00-06:00  2元  (片段1: 05:00-06:30 → 60分钟完整单元)
06:00-06:30  2元  (片段1: 剩余30分钟，收全额)
06:30-07:30  0元  (片段2: 免费时段精确覆盖)
07:30-08:15  2元  (片段3: 07:30-10:00 → 45分钟，收全额)
─────────────────
总计：6元
```

**特点**：
- 在免费时段边界精确切分时间轴
- 切分后的每个片段从片段起点开始按单元长度划分
- 不足一个单元也收全额
- 封顶后截止，剩余时间合并为一个免费单元

## 核心差异

| 维度 | UNIT_BASED | CONTINUOUS |
|------|------------|------------|
| 单元对齐 | 固定从计费起点对齐 | 在免费边界切分后从片段起点重新计时 |
| 免费判断 | 必须完全覆盖整个单元 | 精确覆盖时间段 |
| 片段拆分 | 无 | 切分后超过单元长度的片段继续按单元拆分 |
| 封顶处理 | 按比例削减每个收费单元 | 单价-超出金额，剩余时间合并为免费单元 |
| 周期封顶 | 按周期独立计算 | 按周期独立计算 |

## 免费时段前置条件

参数中的免费时间段满足以下条件：
- 无重合（若检测到重合则抛出异常）
- 按开始时间正序排列
- 相邻免费时段不合并，各自独立作为切分边界

## 封顶行为详解

### UNIT_BASED 封顶

```
场景：停车 05:00 - 12:00，单元60分钟每单元2元，封顶10元

原始：14元 → 按比例削减每个收费单元
结果：每个收费单元金额 × (10/14)
```

### CONTINUOUS 封顶

```
场景：停车 05:00 - 12:00，单元60分钟每单元2元，封顶7元

05:00-06:00  2元
06:00-07:00  2元
07:00-08:00  2元
08:00-09:00  1元  (单价2元 - 超出1元 = 1元，累计达到封顶7元)
09:00-12:00  0元  (封顶截止，合并为免费单元)
─────────────────
总计：7元
```

**封顶计算方式**：
- 当累计金额 + 下一个单元单价 > 封顶值时
- 该单元收取：封顶值 - 已累计金额
- 之后所有单元合并为一个免费单元

**封顶截止边界**：封顶后截止到以下任意一个（先碰到哪个用哪个）：
- 计费结束时间
- 24小时周期结束时间
- 下一个免费时间段

**免费时段后继续免费**：封顶截止后遇到免费时段，免费时段结束后继续生成免费单元直至计费结束或周期结束。

## CONTINUOUS 模式算法详解

### 算法步骤

```
1. 获取计费时间范围 [calcBegin, calcEnd] 和免费时段列表

2. 按免费时段边界切分时间轴：
   a. 输入：免费时段列表（已排序、无重合）
   b. 输出：时间片段列表
   c. 每个免费时段的 beginTime 和 endTime 都作为切分点

3. 对每个时间片段：
   a. 如果片段恰好匹配某个免费时段 → 标记为免费
   b. 否则从片段起点开始按单元长度划分计费单元
   c. 片段内超过单元长度的部分继续拆分
   d. 不足单元长度收全额

4. 按周期应用封顶：
   a. 累计收费金额
   b. 当累计金额即将超过封顶值时，计算最后一个收费单元的金额
   c. 封顶后的所有单元合并为一个免费单元

5. 汇总结果
```

### 示例

**场景**：停车 05:00 - 10:00，单元60分钟，免费时段 06:30-07:30

```
步骤1：切分时间轴
  片段1：05:00 - 06:30（免费时段前）
  片段2：06:30 - 07:30（免费时段）
  片段3：07:30 - 10:00（免费时段后）

步骤2：对每个片段生成计费单元
  片段1 (05:00-06:30, 90分钟)：
    05:00-06:00  2元  (60分钟完整单元)
    06:00-06:30  2元  (30分钟，收全额)

  片段2 (06:30-07:30)：
    06:30-07:30  0元  (免费)

  片段3 (07:30-10:00, 150分钟)：
    07:30-08:30  2元  (60分钟完整单元)
    08:30-09:30  2元  (60分钟完整单元)
    09:30-10:00  2元  (30分钟，收全额)

总计：10元
```

## 已知现象

**现象**：CONTINUOUS 模式下，某些场景有无免费时段费用相同。

**示例**：
```
停车 05:00 - 10:00，单元60分钟，免费时段 06:30-07:30

无免费时段：
05:00-06:00, 06:00-07:00, 07:00-08:00, 08:00-09:00, 09:00-10:00
总计：10元

有免费时段（CONTINUOUS）：
05:00-06:00(2元), 06:00-06:30(2元), 06:30-07:30(0元), 07:30-08:30(2元), 08:30-09:30(2元), 09:30-10:00(2元)
总计：10元
```

**原因**：每个片段从片段起点重新按单元划分，不足单元收全额，导致免费时段切分产生的"碎块"仍然收取全额费用。

**结论**：这是 CONTINUOUS 模式的正常行为，符合"不足单元收全额"的规则。
